# 產品需求文件 (PRD) - 台中大眾運輸交通整合app

## 1. 專案概述
### 背景與動機
台中市的大眾運輸系統包含了公車、捷運、YouBike 以及台鐵等多種運具。目前這些運具的資訊散落在不同的應用程式中，造成使用者常需花費大量時間去了解交通工具的位置動向。本專案的動機在於解決此痛點，希望能讓使用者在日常生活當中，減少花費大量時間了解交通工具位置動向，讓通勤更有效率。透過開發一款整合台中市各種大眾交通工具即時資訊的 Web 應用程式，提供一站式的路線規劃與即時動態掌握，帶來更順暢的交通體驗。

### 目標用戶
- **通勤族**：每天依賴大眾運輸上下班/上學的學生與上班族。
- **觀光客**：對台中市交通不熟悉，需要詳細轉乘指引的旅客。
- **偶爾搭乘者**：平時以汽機車代步，偶爾需要搭乘大眾運輸的市民。

### 核心價值主張
- **一站式整合**：整合台中市境內所有主要大眾運輸系統資訊。
- **無縫轉乘體驗**：提供跨運具的路線與時間規劃，降低轉乘等待時間。
- **即時動態掌握**：提供公車進站、捷運到站、YouBike 剩餘車輛等即時動態資訊。

## 2. 功能需求

### 1. 即時動態查詢
- 可查詢公車、捷運時刻與即時進站資訊，以及 YouBike 站點剩餘車輛數。
- **使用者故事**：作為**通勤族**，我希望**能查詢公車和捷運的即時進站資訊**，以便**我能準確掌握出門時間，避免苦等或錯過班車。**

### 2. 跨運具路線規劃
- 輸入起訖點，系統自動建議包含不同運具組合的最佳路線（最短時間、最少轉乘、最少步行等）。
- **使用者故事**：作為**觀光客**，我希望**輸入出發地和目的地後能得到詳細的轉乘路線**，以便**我能順利抵達景點而不必擔心迷路。**

### 3. 常用站牌與路線收藏
- 使用者可以儲存每天搭乘的路線或站牌，方便快速查看。
- **使用者故事**：作為**通勤族**，我希望**能夠收藏我常搭的公車站牌**，以便**我每天早上能快速確認公車還要多久才到。**

### 4. 票價與時間估算
- 提供所選路線的預估抵達時間及總票價。
- **使用者故事**：作為**精打細算的使用者**，我希望**系統能估算不同路線的總票價**，以便**我選擇最經濟實惠的搭乘方式。**

### 5. 最新交通營運公告
- 整合台中市交通局或客運業者的臨時改道、停駛或捷運營運狀況公告。
- **使用者故事**：作為**所有使用者**，我希望**能看到最新的交通營運公告**，以便**在遇到突發狀況或改道時能提早應變。**

## 3. 非功能需求

### 技術限制
- **Backend (後端)**：使用 Python Flask 框架開發。
- **Frontend (前端)**：使用 Jinja2 模板引擎進行畫面渲染，搭配 HTML/CSS/JavaScript。
- **Database (資料庫)**：使用 SQLite 儲存使用者資料與收藏記錄。

### 效能與安全考量
- **效能**：即時動態資料需串接外部 API (如 TDX 交通資料服務) 或進行快取處理，以降低頁面載入時間與 API 請求次數，確保回應時間低於 3 秒。
- **安全性**：
  - 使用者密碼需經過雜湊處理後儲存。
  - 需要防止常見的 Web 攻擊，如 SQL Injection 與 XSS。
  - 確保使用者個人資料與收藏紀錄的安全。
