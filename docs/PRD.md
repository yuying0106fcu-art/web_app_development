# 產品需求文件 (PRD) - 食譜收藏夾系統

## 1. 專案概述
### 背景與動機
現代人生活忙碌，常為了三餐不知道要吃什麼、買什麼菜而困擾。許多人在網路上看到喜歡的食譜會先儲存下來，但往往散落在不同的平台（如 IG、FB、網頁書籤）難以統整。本專案旨在打造一個專屬的「食譜收藏夾系統」，幫助使用者集中管理食譜，並結合購物清單、互動烹飪與預算考量，讓下廚變得更輕鬆、更有效率。

### 目標用戶
- **一般用戶/懶人/外宿族**：希望有簡單易懂的食譜，並能快速知道需要買哪些食材。
- **家庭煮夫/煮婦**：需要規劃預算與搭配超市優惠，並有效管理家庭常備菜單。
- **系統管理者**：負責維護系統運作、管理公用食譜資料庫及用戶帳號。

### 核心價值主張
一個高度整合的食譜管家，從「決定吃什麼」、「採買清單」、「控制預算」到「跟著做」，提供一站式、無痛下廚的完美體驗。

## 2. 功能需求

### 1. 標籤和分類系統
- 允許使用者自訂多維度的標籤（如：中式、西式、甜點、晚餐、低卡）來組織食譜。
- **使用者故事**：作為**一般使用者**，我希望**能幫收藏的食譜加上多個標籤與分類**，以便**我在想煮特定種類的料理時能快速找到對應食譜。**

### 2. 一鍵生成購物清單
- 系統能自動擷取食譜中的食材和份量，合併總結成一份購物清單。
- **使用者故事**：作為**家庭煮夫/煮婦**，我希望**能選擇未來幾天的食譜並一鍵產生購物清單**，以便**我去超市採買時不會漏買或多買食材。**

### 3. 互動式烹飪模式
- 提供專心下廚的 UI 模式，字體放大、步驟分頁顯示，甚至支援語音或手勢翻頁，並內建計時器。
- **使用者故事**：作為**一般使用者**，我希望**能在做菜時開啟互動式烹飪模式**，以便**我手濕或髒的時候，仍能輕鬆且清楚地看著步驟跟著做。**

### 4. 預算與超市優惠整合
- 允許使用者為清單設定預算，或輸入食材大約市價，系統可計算該餐成本。若有整合公開的超市優惠資訊，能給出採買建議。
- **使用者故事**：作為**精打細算的使用者**，我希望**能看到這份食譜或採買清單的預估費用**，以便**我能更好地控制當月伙食費。**

### 5. 外宿/懶人專用模式
- 提供「微波爐專用」、「電鍋料理」、「一鍋到底」、「5分鐘出餐」等快速篩選條件。
- **使用者故事**：作為**外宿學生或懶人**，我希望**能開啟外宿懶人專用模式**，以便**我只看到那些步驟極簡、不需複雜廚具就能完成的食譜。**

## 3. 非功能需求

### 技術限制
- **Backend (後端)**：使用 Python Flask 框架開發。
- **Frontend (前端)**：使用 Jinja2 模板引擎進行畫面渲染，搭配 CSS/JS 提升互動性。
- **Database (資料庫)**：使用 SQLite 儲存使用者資料、食譜內容、標籤及清單。

### 效能與安全考量
- **效能**：頁面載入時間應在 2 秒內完成，標籤搜尋與一鍵生成清單的反應時間需小於 1 秒。
- **安全性**：
  - 使用者密碼需經過雜湊處理（如 PBKDF2 或 bcrypt）後儲存。
  - 需要防止常見的 Web 攻擊，如 SQL Injection（透過 ORM 或 Parameterized queries 處理）與 XSS（防範使用者輸入惡意腳本）。
  - 各別用戶食譜內容有權限控管機制，防止未授權存取他人私有食譜。

## 4. MVP 範圍

| 功能 | 優先級 | 說明 |
| --- | --- | --- |
| **食譜 CRUD (新增/讀取/更新/刪除)** | Must Have | 基本食譜建立與管理（含標題、食材、步驟）。 |
| **標籤和分類系統** | Must Have | 食譜可以自訂標籤並支援標籤檢索。 |
| **一鍵生成購物清單** | Must Have | 自動將選定的食譜食材統整成清單。 |
| **使用者註冊與登入** | Must Have | 以區隔不同使用者的個人化收藏。 |
| **外宿/懶人專用模式** | Should Have | 提供專屬分類入口或快速篩選器。 |
| **互動式烹飪模式** | Should Have | 排版易讀的大字體模式，提供下廚時良好的閱讀體驗。 |
| **預算與超市優惠整合** | Nice to Have | 初期可先提供讓使用者手動輸入估價的功能，未來迭代再串聯外部優惠服務。 |
